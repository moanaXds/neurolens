## вңҰ What is NeurЕҚLens?

NeurЕҚLens is a **multi-modal vector search system** that transforms raw images and natural language into high-dimensional embeddings using OpenAI's **CLIP model** вҖ” then stores, indexes, and retrieves them via **Pinecone** and **MongoDB Atlas Vector Search**.

No supervised training. No labeled pipelines. Just embeddings and geometry.

```
You give it: "wolf in snow"
It returns:  рҹҗәрҹҢЁпёҸ вҖ” the closest matching images in the dataset
```

## вңҰ Tasks

<details>
<summary><b>рҹ”Қ Task 1 вҖ” Image Similarity Search</b></summary>

<br/>

**Goal:** Given an image, retrieve the most visually similar images from the dataset.

**How it works:**
- Every image is passed through CLIP's vision encoder вҶ’ produces a `512-d` float vector
- All vectors are stored in Pinecone / MongoDB Atlas
- A query image is encoded вҶ’ cosine similarity search is performed
- Top-K nearest neighbors are returned and displayed

**Expected Output:**

```
Query Image: dog_01.jpg
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
Top 5 Similar Images:
  [1] dog_07.jpg     similarity: 0.97
  [2] dog_03.jpg     similarity: 0.95
  [3] dog_11.jpg     similarity: 0.93
  [4] wolf_02.jpg    similarity: 0.81
  [5] cat_04.jpg     similarity: 0.74
```

</details>

<details>
<summary><b>рҹҸ·пёҸ Task 2 вҖ” Zero-Shot Classification via KNN</b></summary>

<br/>

**Goal:** Classify an unknown image without training a classifier.

**How it works:**
- Each stored vector has an associated label (`cat`, `dog`, `wolf`, etc.)
- For a new query image, retrieve top-K nearest neighbors
- Apply majority voting over neighbor labels вҶ’ predicted class

**Expected Output:**

```
Input Image: unknown_animal.jpg

Nearest Neighbors:
  wolf_04.jpg  вҶ’ wolf   (dist: 0.12)
  wolf_09.jpg  вҶ’ wolf   (dist: 0.14)
  dog_02.jpg   вҶ’ dog    (dist: 0.21)
  wolf_01.jpg  вҶ’ wolf   (dist: 0.23)
  wolf_07.jpg  вҶ’ wolf   (dist: 0.25)

Votes: wolf=4, dog=1
Predicted Label: рҹҗә wolf
```

</details>

<details>
<summary><b>рҹ’¬ Task 3 вҖ” Cross-Modal Search (Text вҶ’ Image)</b></summary>

<br/>

**Goal:** Search images using plain English text queries.

**How it works:**
- CLIP's text encoder converts a string query into a `512-d` vector
- This lives in the **same embedding space** as image vectors
- Cosine similarity retrieves the best image matches for the text

**Example Queries:**
```
"wolf in snow"    вҶ’ returns snowy wolf images
"cute puppy"      вҶ’ returns close-up dog images  
"black cat"       вҶ’ returns dark-colored cat images
```

**This is why CLIP is powerful** вҖ” the model was trained on 400M image-text pairs, so text and image semantics are aligned.

</details>

## вңҰ Key Concepts

**Why CLIP?**
CLIP (Contrastive LanguageвҖ“Image Pretraining) was trained to align image and text representations. This means `encode("a dog")` and `encode(dog_image.jpg)` produce geometrically close vectors вҖ” enabling cross-modal retrieval with zero task-specific training.

**Why Vector Databases?**
Traditional SQL `WHERE` clauses can't find "similar" things. Vector DBs use Approximate Nearest Neighbor (ANN) algorithms вҖ” like HNSW or IVFFlat вҖ” to find the closest vectors in milliseconds, even across millions of entries.

**Cosine Similarity:**
```
similarity(A, B) = (A В· B) / (||A|| Г— ||B||)
```
Range: `[-1, 1]` вҶ’ closer to `1` = more similar.

## вңҰ Pinecone vs MongoDB Atlas вҖ” Comparison

| Feature | Pinecone | MongoDB Atlas |
|---|---|---|
| Type | Purpose-built vector DB | Multi-model DB + vector search |
| Setup | Fully managed, API-first | Atlas cluster + index config |
| Query Speed | Extremely fast (dedicated ANN) | Fast, but secondary to doc ops |
| Metadata Filtering | вң… Native | вң… via `$match` pipeline |
| Best For | Pure vector workloads | Apps needing vectors + documents |

## вңҰ Observations

- CLIP embeddings are **surprisingly robust** вҖ” text queries like `"wolf in snow"` retrieve contextually correct images without any fine-tuning
- KNN classification via vector search **matches simple CNN accuracy** on clean datasets with no training overhead
- Pinecone offers faster cold-start indexing; MongoDB is preferable when your application already stores structured data
- Cross-modal search degrades on **ambiguous text** вҖ” `"animal"` returns scattered results across all classes
